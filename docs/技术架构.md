# FactorAI 技术架构（V1）

## 1. 架构总览

V1 采用前端优先架构，先完成可运行产品原型与交互闭环，再逐步接入后端服务。

- 前端：React + TypeScript + Vite
- 路由：React Router
- 可视化画布：React Flow
- 图标：Lucide

## 2. 前端模块划分

### 2.1 页面层

- 首页（品牌入口）
- 工作流列表页
- 模板库页
- 编辑器页
- 运行中心页
- 研究报告页

### 2.2 组件层

- AppShell（主布局）
- 侧边导航
- 顶部工具栏
- 数据卡片与列表组件
- 编辑器的节点库/画布/属性面板

### 2.3 数据层

- `mock.ts` 提供工作流、模板、节点、运行记录、报告指标
- 后续替换为 API SDK（保持接口字段兼容）

## 3. 后端预留接口（下一阶段）

- `GET /api/workflows`
- `POST /api/workflows`
- `POST /api/runs`
- `GET /api/runs/:id`
- `GET /api/reports/:workflowId`
- `POST /api/nodes/execute`（Python 节点沙箱执行）

## 4. Python 节点执行预案

- 输入：`df_input`, `context`, `params`
- 输出：`df_factor` 或 `model_obj + metrics`
- 安全策略：超时、内存限制、包白名单、危险语句拦截

## 5. 可扩展性设计

- 节点采用 schema 描述，支持动态注册
- 画布与节点配置解耦，便于后续切换执行引擎
- 报告组件化，支持接入真实回测结果
