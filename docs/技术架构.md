# FactorAI 技术架构（V1）

## 1. 架构总览

当前版本采用前后端分离架构，已实现可部署的完整闭环。

- 前端：React + TypeScript + Vite
- 路由：React Router
- 可视化画布：React Flow
- 后端：FastAPI + SQLModel + SQLite
- 执行引擎：DAG 拓扑执行 + 节点处理器 + 异步任务线程池
- Python 节点：多进程沙箱（超时、内存限制、输出字段校验）

## 2. 前端模块划分

### 2.1 页面层

- 首页（品牌入口）
- 工作流列表页
- 模板库页
- 编辑器页
- 运行中心页
- 研究报告页

### 2.2 组件层

- AppShell（主布局）
- 侧边导航
- 顶部工具栏
- 数据卡片与列表组件
- 编辑器的节点库/画布/属性面板

### 2.3 数据层

- 前端 `api/client.ts` 对接后端 API
- 前端 `WorkspaceContext` 管理状态与轮询刷新
- 后端 SQLite 持久化 workflows/templates/runs/reports

## 3. 已实现后端接口

- `GET /api/workflows`
- `POST /api/workflows`
- `PUT /api/workflows/{id}/graph`
- `POST /api/workflows/{id}/draft`
- `POST /api/runs`
- `POST /api/workflows/{id}/run`
- `GET /api/runs/:id`
- `GET /api/reports/:workflowId`
- `GET /api/templates`
- `POST /api/templates/{id}/clone`
- `GET /api/node-library`

## 4. Python 节点执行方案（已落地）

- 输入：`df_input` DataFrame
- 输出：必须包含 `date/symbol/factor1` 的 `df_factor`
- 安全策略：多进程执行 + 超时终止 + RLIMIT 内存限制 + 安全内建函数集

## 5. 可扩展性设计

- 节点采用 schema 描述，支持动态注册
- 画布与节点配置解耦，便于后续切换执行引擎
- 报告组件化，支持接入真实回测结果
