# 节点协议规范（V1）

## 1. 节点定义结构

```json
{
  "id": "factor.pca",
  "name": "PCA因子构建",
  "category": "06-线下专属",
  "description": "基于特征重要性构建复合因子",
  "inputs": ["features", "label"],
  "outputs": ["factor_score", "weight"],
  "params": [
    { "key": "components", "type": "number", "default": 5 }
  ]
}
```

## 2. 输入输出约束

- 所有表格型数据采用 DataFrame 语义
- 必备字段：`date`, `symbol`
- 因子输出必须包含：`date`, `symbol`, `factor1`

## 3. Python 节点约束

### 3.1 输入变量

- `df_input`: pandas.DataFrame
- `context`: `{ start_date, end_date, run_id }`
- `params`: 节点参数字典

### 3.2 输出变量

- 因子场景：`df_factor`
- 模型场景：`model_obj`, `metrics`

### 3.3 失败判定

- 缺失输出变量
- 输出字段不完整
- 运行超时或异常退出

## 4. 质量校验

- 空值比例阈值（默认 30%）
- 重复键检查（`date + symbol`）
- 日期升序检查

## 5. 日志规范

- `INFO`：参数、输入规模、耗时
- `WARN`：数据质量问题
- `ERROR`：异常堆栈与失败原因
