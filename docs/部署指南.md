# FactorAI 部署指南（服务器算力版本）

本文档面向“自有服务器作为算力资源”的部署场景。

## 1. 部署方式 A：Docker Compose（推荐）

### 1.1 前置条件

- Linux 服务器（建议 4C8G 起步）
- 安装 Docker / Docker Compose

### 1.2 一键启动

```bash
cd /path/to/FactorAI
docker compose up -d --build
```

启动后：

- 前端：http://服务器IP/
- 后端 API：http://服务器IP:8000/api/health

### 1.3 停止与重启

```bash
docker compose down
docker compose up -d
```

## 2. 部署方式 B：源码方式

### 2.1 后端

```bash
cd server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 2.2 前端

```bash
cd web
npm install
VITE_API_BASE_URL=http://服务器IP:8000/api npm run build
npm run preview -- --host 0.0.0.0 --port 5173
```

## 3. 算力与任务建议

- `max_workers=4` 可在 `server/app/services/execution.py` 中调整
- Python 节点超时与内存限制：
  - `FACTORAI_PYTHON_NODE_TIMEOUT`
  - `FACTORAI_PYTHON_NODE_MEMORY_MB`
- 生产建议将 SQLite 升级为 PostgreSQL（可平滑迁移）

## 4. 运维建议

- 使用 Nginx/Traefik 做统一入口与 HTTPS
- 定时备份 `factorai.db`
- 对 `/api/workflows/*/run` 做鉴权和限流
